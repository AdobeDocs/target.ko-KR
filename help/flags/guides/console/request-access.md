# 액세스 요청 {#request-access}

플래그에 대한 액세스는 팀 수준에서 관리됩니다. 플래그를 사용하려면 적절한 역할을 가진 팀에 추가되어야 합니다.

## 사전 요구 사항 {#prerequisites}

액세스를 요청하기 전에 다음을 확인하십시오.

* 어느 팀에 합류할지 알잖아. 제품 또는 엔지니어링 리드에 문의하여 팀과 해당 관리자를 확인하십시오.
* 액세스해야 하는 환경(**단계**, **프로덕션** 또는 둘 다)을 알고 있습니다.

## 액세스 요청 {#request-steps}

1. 조직의 액세스 관리 시스템으로 이동합니다.
2. **플래그**&#x200B;를 검색하고 환경(스테이지 또는 프로덕션)에 적합한 응용 프로그램을 선택합니다.
3. 팀 이름을 검색하고 요청하는 역할을 선택합니다.
4. 요청을 제출합니다.

요청이 제출되면 확인을 받게 됩니다. **보류 중** 요청의 액세스 관리 시스템에서 요청 상태를 모니터링할 수 있습니다.

>[!NOTE]
>
>팀에서 **관리자** 또는 **기능 관리자** 역할을 가진 구성원은 액세스 요청을 승인하거나 거부할 수 있습니다.

## 사용 가능한 역할 {#roles}

사용자에게 할당된 역할에 따라 콘솔에서 수행할 수 있는 작업이 결정됩니다. 요청할 역할을 잘 모를 경우 팀 관리자에게 문의하십시오.

## 액세스 권한 부여 후 {#after-access}

요청이 승인되면 Adobe Target 내에서 플래그 를 열고 팀과 애플리케이션이 표시되는지 확인합니다. 다음 단계는 [Adobe Target에서 플래그 액세스](log-in-to-the-console.md)를 참조하십시오.

## 도움이 필요하십니까? {#need-help}

가입할 팀을 잘 모르거나 프로젝트에 대한 팀이 아직 없는 경우 조직의 플래그 관리자에게 문의하십시오.
