# 액세스 요청 {#request-access}

플래그에 대한 액세스는 팀 수준에서 관리됩니다. 플래그를 사용하려면 적절한 역할을 가진 팀에 추가되어야 합니다.

## 사전 요구 사항 {#prerequisites}

액세스를 요청하기 전에 다음을 확인하십시오.

* 어느 팀에 합류할지 알잖아. 제품 또는 엔지니어링 리드에 문의하여 팀과 해당 관리자를 확인하십시오.
* 액세스 권한이 필요한 샌드박스를 알 수 있습니다.

## 액세스 요청 {#request-steps}

액세스를 요청하려면 Adobe 담당자에게 문의하십시오.

## 사용 가능한 역할 {#roles}

사용자에게 할당된 역할에 따라 콘솔에서 수행할 수 있는 작업이 결정됩니다. 요청할 역할을 잘 모를 경우 관리자에게 문의하십시오.

| 역할 | 설명 |
| --- | --- |
| **관리** | 콘솔에 대한 전체 액세스 권한. 애플리케이션 관리 가능 |
| **제품 릴리스 소유자** | 기능 플래그와 기능 그룹을 만들고 관리할 수 있으며 외부 사용자에게 기능을 릴리스할 수 있습니다. |

## 액세스 권한 부여 후 {#after-access}

요청이 승인되면 Adobe Target 내에서 플래그 를 열고 애플리케이션이 표시되는지 확인합니다. 다음 단계는 [Adobe Target에서 플래그 액세스](log-in-to-the-console.md)를 참조하십시오.

## 도움이 필요하십니까? {#need-help}

가입할 팀을 잘 모르거나 프로젝트에 대한 팀이 아직 없는 경우 조직의 플래그 관리자에게 문의하십시오.
