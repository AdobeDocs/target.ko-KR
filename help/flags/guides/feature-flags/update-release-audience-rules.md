# 릴리스 대상 규칙 업데이트 {#update-release-audience-rules}

## 대상자 설정에 액세스 {#access}

릴리스를 수신하는 사용자를 구성하려면 콘솔의 대상 설정으로 이동합니다.

1. 플래그 콘솔에서 릴리스를 엽니다.
2. **대상자** 탭으로 이동합니다.
3. **대상 규칙** 옆에 있는 토글을 활성화합니다.
4. **포함 규칙**(릴리스를 받을 사용자) 또는 **제외 규칙**(그렇지 않은 사용자)을 추가할지 여부를 선택합니다.

## 지원되는 대상 기준 {#criteria}

### ID 유형 {#id-type}

계정 유형(예: 개인 또는 기업 계정)을 기반으로 사용자를 타깃팅합니다.

### 국가 {#country}

등록된 국가를 기준으로 사용자를 타겟팅합니다.

### 사용자 ID {#user-id}

고유한 사용자 식별자(GUID)로 특정 사용자를 타깃팅합니다.

### 이메일 주소 {#email}

정확한 이메일 주소로 사용자를 타겟팅합니다. 여러 전자 메일 주소를 한 번에 추가하려면 연산자 드롭다운에서 **in**&#x200B;을(를) 선택하고(**is** 대신) 다음을 수행합니다.

* 텍스트 상자에 쉼표로 구분된 주소 목록을 입력하거나
* 한 줄에 하나의 주소가 있는 `.txt` 파일을 업로드하거나
* 주소 목록이 있는 `.csv` 파일 업로드

목록이 매우 큰 경우 더 작은 배치로 분할합니다.

### 이메일 도메인 {#email-domain}

전자 메일 주소가 특정 도메인에 속한 모든 사용자를 타겟팅합니다(예: `yourcompany.com`).

### 클라이언트 IP {#client-ip}

클라이언트 IP 주소를 기반으로 한 타겟 사용자입니다.

### 백분율 {#percentage}

인증되지 않은 사용자를 포함하여 모든 사용자의 임의 백분율을 타깃팅합니다.

### 백분율(인증만) {#percentage-auth}

인증된 사용자의 무작위 백분율만 타깃팅합니다. 인증되지 않은 사용자는 제외됩니다.

## 여러 규칙 결합 {#combining-rules}

AND/OR 논리를 사용하여 여러 대상 기준을 결합할 수 있습니다.

**예:**

* 특정 도메인의 엔터프라이즈 계정만 대상으로 지정하려면 AND를 사용하여 **ID 유형** 규칙을 **전자 메일 도메인** 규칙과 결합하십시오.
* 특정 국가의 사용자 10%를 타깃팅하려면 **국가** 규칙과 **백분율** 규칙을 결합하십시오.

**+/-** 아이콘을 사용하여 조건을 추가하거나 제거하십시오. 조건을 복제하려면 기존 규칙 옆에 있는 복제 아이콘을 선택합니다. 복잡한 중첩 논리의 경우 **중첩 논리**&#x200B;를 사용하도록 설정하고 텍스트 상자에 올바른 식을 입력하십시오.
