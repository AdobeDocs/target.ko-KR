# 기능 그룹 만들기 {#create-feature-group}

## 사전 요구 사항 {#prerequisites}

피쳐 그룹을 생성하기 전에 다음을 완료합니다.

* 플래그 콘솔에 액세스할 수 있습니다. [콘솔에 로그인](../console/log-in-to-the-console.md)을 참조하세요.
* 응용 프로그램이 온보딩되었습니다. [응용 프로그램 온보딩](../applications/onboard-your-application.md)을 참조하십시오.
* **개발자** 또는 **제품 릴리스 소유자** 역할이 있습니다.
* 그룹에 추가할 기능 플래그를 만들었습니다. [첫 번째 기능 플래그 만들기](create-your-first-feature-flag.md)를 참조하십시오.

기능 그룹을 소개하려면 [여러 기능을 제어할 기능 그룹](../../concepts/feature-groups-to-control-multiple-features.md)을 참조하세요.

## 1단계: 기능 그룹 만들기 {#create}

콘솔을 열고 새 기능 그룹을 시작합니다.

1. 플래그 콘솔에 로그인하고 **기능 테스트 > 기능 그룹**(으)로 이동합니다.
2. **새 기능 그룹**&#x200B;을(를) 선택하십시오.

## 2단계: 기본 세부 정보 {#basic-details}

기능 그룹에 대한 일반 설정을 구성합니다.

1. 제목, 키, 설명 및 선택적으로 태그를 제공합니다.
2. 기능 그룹에 대해 **백분율 롤아웃**&#x200B;을 설정합니다.
3. A/B 테스트를 실행하려면 변형을 두 개 이상 선택합니다. 그렇지 않으면 한 가지 변형에 둡니다. 자세한 내용은 [기능 플래그를 사용하여 A/B 테스트](a-b-testing.md)를 참조하십시오.

## 3단계: 대상 {#audience}

이 그룹의 기능을 받을 사용자를 정의합니다.

1. **대상** 탭에서 대상 기준을 추가하여 기능을 수신할 사용자를 정의합니다.
2. **응용 프로그램**&#x200B;에서 팀의 응용 프로그램을 하나 이상 추가합니다. 기능 그룹은 모두 동일한 팀에 속하는 한 여러 애플리케이션에 걸쳐 있을 수 있습니다.

>[!NOTE]
>
>**개발자** 역할이 샌드박스에 추가되었습니다. **대상 > 프로필 > 사용자 ID** 아래에 자신의 사용자 ID를 추가하여 비공개로 테스트하십시오. 외부 사용자를 타깃팅하려면 **제품 릴리스 소유자** 역할이 필요합니다.

## 4단계: 기능 {#features}

이 그룹에서 제어할 기능 플래그를 할당하십시오.

1. **기능** 탭 아래에 선택한 각 응용 프로그램에 대한 상자가 표시됩니다.
2. 각 애플리케이션에 대한 기능 플래그를 추가합니다.
3. A/B 테스트를 위해 여러 변형을 선택한 경우 각 변형에 대한 탭이 표시됩니다. 각 기능에 적절한 기능 플래그를 추가합니다.

>[!IMPORTANT]
>
>기능 플래그는 기능 플래그, 기능 그룹 또는 릴리스를 통해 한 가지 방법으로만 제공될 수 있습니다. 기능 플래그를 기능 그룹에 추가하면 해당 플래그에 설정된 대상자 또는 백분율 롤아웃이 제거됩니다. 다른 릴리스 또는 기능 그룹에 이미 할당된 기능 플래그는 목록에 표시되지 않습니다.

>[!IMPORTANT]
>
>기능 그룹에서 기능 플래그를 **제거**&#x200B;할 때 플래그가 **사용 안 함** 상태로 돌아가고 대상이 **복원되지 않음**&#x200B;입니다. 새 플래그로 처리하십시오. 그룹 내의 **disabled** 플래그는 항상 `false`로 평가됩니다. 기능 그룹을 활성화하면 구성원 플래그를 활성화하지 **않습니다**. 각 플래그를 명시적으로 활성화하십시오.
>
>기능 그룹은 **관리 계층**&#x200B;입니다. 런타임 시 항상 **기능(플래그) 수준**&#x200B;에서 평가하고 그룹 수준에서는 평가하지 않습니다. 응답에는 사용자가 속한 변형이 포함됩니다.

## 5단계: 예약(선택 사항) {#schedule}

기능 그룹 설정에서 **예약** 옵션을 사용하여 향후 날짜 및 시간에 기능 그룹을 활성화하도록 예약할 수 있습니다.
