# 전체 워크플로 릴리스 {#release-workflow}

이 페이지에서는 Release Manager에서 관리하는 조정된 릴리스와 관련된 작업의 전체 시퀀스에 대해 설명합니다.

## &#x200B;1. 애플리케이션별 기능 플래그 정의 {#define-flags}

각 제품 팀은 플래그 콘솔에 로그인하고 소유한 응용 프로그램의 기능 플래그를 만드는 **제품 릴리스 소유자**&#x200B;를 할당합니다. 그런 다음 제품 팀은 이러한 플래그 뒤에 기능을 배치하여 코드에 조건부 논리를 구현합니다.

## &#x200B;2. 릴리스 만들기 {#create-release}

새 릴리스를 생성하려면 지원 요청이 필요합니다. 완전히 셀프 서비스는 아닙니다. 지원팀에 문의하여 릴리스를 생성하십시오. 릴리스 이름, 소유 팀, 대상 환경, 목표, 참여 애플리케이션 및 예상 기간을 제공합니다.

릴리스가 확인되면 릴리스 관리자가 콘솔을 열고 릴리스 구성을 완료합니다.

## &#x200B;3. 릴리스에 기능 플래그 추가 {#add-flags}

릴리스에 추가된 각 애플리케이션에 대해 해당 애플리케이션의 제품 릴리스 소유자가 로그인하여 관련 기능 플래그를 릴리스에 추가합니다. 이러한 플래그는 릴리스와 함께 사용할 기능을 나타냅니다.

## &#x200B;4. 대상자 구성 {#configure-audience}

Release Manager는 릴리스에 대한 대상(예: 특정 국가의 사용자 중 1%, 특정 이메일 도메인을 가진 모든 사용자 또는 특정 사용자 ID 목록)을 선택합니다. 구성이 완료되면 Release Manager는 릴리스를 저장하고 게시하여 지정된 대상에 대해 라이브로 푸시합니다.

## &#x200B;5. 롤아웃 확장 및 관리 {#expand}

릴리스가 라이브로 전환되면 릴리스 관리자는 대상자 규칙을 조정하여 롤아웃을 점차적으로 확장하고 문제를 모니터링하며 릴리스 상태 컨트롤을 사용하여 라이프사이클을 관리할 수 있습니다. 자세한 내용은 [릴리스 상태](release-states.md)를 참조하십시오.

## 주요 사항 {#key-points}

* 모든 참여 응용 프로그램은 기능 플래그 뒤에 코드를 독립적으로 프로덕션에 배포하므로 조정된 배포 시간이 필요하지 않습니다.
* 릴리스는 팀 간의 모든 플래그를 동시에 활성화하는 단일 제어 지점입니다.
* 릴리스에 대한 대상 타깃팅은 인증 토큰 특성(국가, 이메일, 사용자 ID, 백분율)을 기반으로 합니다.
