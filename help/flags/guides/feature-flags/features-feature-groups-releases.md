# 기능 및 기능 그룹 {#features-feature-groups}

플래그는 기능 롤아웃 관리를 위한 두 가지 아티팩트를 제공합니다. 올바른 옵션을 선택하는 것은 롤아웃 범위와 관련된 기능의 수에 따라 다릅니다.

## 두 가지 아티팩트 {#artifacts}

**기능 플래그**
가장 작은 단위입니다. 단일 응용 프로그램에 대한 단일 기능을 제어합니다. 정의된 대상에 대해 활성화하거나 비활성화할 수 있습니다.

**기능 그룹**
동일한 팀에 속한 기능 플래그의 컬렉션입니다. 여러 애플리케이션에서 여러 플래그를 한 단위로 관리할 수 있습니다.

## 비교 {#comparison}

| | 기능 플래그 | 기능 그룹 |
|---|---|---|
| 대상을 관리하는 데 필요한 **역할** | 제품 릴리스 소유자 | 제품 릴리스 소유자 |
| **응용 프로그램** | 단일 | 다중(동일한 팀) |
| **풍부한 대상 기준** | ✓ | ✓ |
| **롤아웃 비율** | 모든 대상 기준과 결합 | 모든 대상 기준과 결합 |
| **A/B 테스트** | ✓ | ✓ |
| **셀프 서비스** | ✓ | ✓ |
| **감사 추적** | ✓ | ✓ |

## 각 항목 사용 시기 {#when-to-use}

| 시나리오 | 사용 |
|---|---|
| 하나의 애플리케이션에 대해 단일 기능 테스트 또는 롤아웃 | **기능 플래그** |
| 동일한 팀에서 여러 기능을 조정하면서 동시에 실행 | **기능 그룹** |
