# 첫 번째 기능 플래그 만들기 {#create-feature-flag}

## 사전 요구 사항 {#prerequisites}

기능 플래그를 만들기 전에 다음을 완료하십시오.

* 플래그 콘솔에 액세스할 수 있습니다. [콘솔에 로그인](../console/log-in-to-the-console.md)을 참조하세요.
* 응용 프로그램이 온보딩되었습니다. [응용 프로그램 온보딩](../applications/onboard-your-application.md)을 참조하십시오.
* **개발자** 또는 **제품 릴리스 소유자** 역할이 있습니다.

## 1단계: 기능 플래그 만들기 {#create}

새 기능 플래그를 만들려면 콘솔에서 다음 단계를 수행하십시오.

1. 플래그 콘솔에 로그인하고 **기능 및 릴리스 > 기능 플래그**(으)로 이동합니다.
1. **응용 프로그램** 드롭다운에서 응용 프로그램을 선택합니다.
1. **새 기능**&#x200B;을 선택하세요.
1. 양식 필드를 작성합니다.

   | 필드 | 설명 |
   | --- | --- |
   | **이름** | 기능 플래그에 대한 표시 레이블입니다. 코드에서 사용되지 않습니다. |
   | **키** * | 코드에서 플래그를 평가하는 데 사용되는 식별자입니다. 만든 후에는 변경할 수 없습니다. |
   | **설명** | 설명서에 사용할 선택적인 설명입니다. |
   | **메타데이터** | 선택 사항입니다. 최대 1,024자. 플래그와 연결할 추가 메타데이터에 대해 이 필드를 사용합니다. |
   | **ID** * | 플래그가 평가되는 ID(예: ECID)입니다. 기능 요청에서 전달된 ID입니다. |
   | **롤아웃 비율** | 이 기능에 제공되는 정의된 대상자의 백분율입니다. 기본값은 100%입니다. [점진적으로 롤아웃할 기능 설정](set-feature-gradual-rollout.md)을 참조하십시오. |

   *로 표시된 필드는 필수입니다.

>[!IMPORTANT]
>
>**Key**&#x200B;은(는) 코드에서 사용되는 식별자이며 만든 후에는 변경할 수 없습니다. 키 **은(는) 공백을 포함할 수 없습니다**. **대/소문자를 구분합니다**. **Name**&#x200B;은(는) 표시 레이블만 사용하며 코드에서 사용되지 않습니다. 두 이름은 독립적입니다(이름은 키로 변환되지 않음). 키 필드에 공백을 입력하면 다음 오류가 발생합니다. _&quot;기능 키에 대한 값이 잘못되었습니다.&quot;_

1. 필요한 경우 대상 기준을 추가합니다(2단계 참조).
1. 기능 플래그 설정을 저장합니다.

## 2단계: 대상 기준 추가 {#audience}

대상 기준은 기능을 볼 사용자를 제어합니다. 기능 요청에서 웹 사이트 또는 앱이 보내는 값(예: `locale` 또는 `platform`)인 **컨텍스트 특성**&#x200B;을 사용하여 사용자를 타깃팅할 수 있습니다. **AND**, **OR** 및 **NOT**&#x200B;과(와) 결합하십시오. [대상 규칙에 컨텍스트 사용](../audience/using-context-in-audience-rules.md)을 참조하십시오.

대상 기준을 추가하려면 기능 플래그를 만들거나 편집할 때 **대상** 탭으로 이동하십시오.

>[!NOTE]
>
>**개발자** 역할이 샌드박스에 추가되었습니다. 개발자는 **대상 > 프로필 > 사용자 ID**&#x200B;에서 자신의 사용자 ID를 추가하여 기능을 노출할 수만 있습니다. 외부 사용자에게 기능을 노출하려면 **제품 릴리스 소유자** 역할이 필요합니다.

## 3단계: 대상 크기 계산 {#audience-size}

대상 기준을 추가한 후 맨 아래 막대에서 **계산**&#x200B;을 선택하여 기능에 적합한 예상 사용자 수를 가져옵니다. 이렇게 하면 라이브로 전환되기 전에 타깃팅을 확인하는 데 도움이 됩니다.

## 4단계: 예약(선택 사항) {#schedule}

기능 플래그 설정에서 **예약** 옵션을 사용하여 향후 날짜 및 시간에 활성화할 기능 플래그를 예약할 수 있습니다.

## FAQ: 기능 플래그를 개발자로 추가할 수 없음 {#faq}

**개발자** 역할이 샌드박스에 추가되었습니다. 개발자는 대상에 사용자 ID를 추가하여 기능을 개인적으로 테스트할 수 있습니다. 외부 사용자에게 기능을 노출할 수 없습니다. **제품 릴리스 소유자** 역할을 사용하여 외부 사용자에게 기능을 릴리스하십시오. 관리자에게 문의하여 역할을 업데이트하십시오.
