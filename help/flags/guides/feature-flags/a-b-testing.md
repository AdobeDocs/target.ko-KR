# 기능 플래그를 사용하여 A/B 테스트 {#a-b-testing}

플래그의 A/B 테스트는 **기능 그룹**&#x200B;을 사용하여 수행됩니다. 기능 그룹에 두 개 이상의 변형을 구성하면 다양한 버전의 기능을 대상의 다양한 하위 집합에 제공하고 결과를 비교할 수 있습니다.

## 사전 요구 사항 {#prerequisites}

* 콘솔에 액세스할 수 있습니다. [콘솔에 로그인](../console/log-in-to-the-console.md)을 참조하세요.
* 팀에 속해 있고 애플리케이션이 온보딩되었습니다.
* **개발자** 또는 **제품 릴리스 소유자** 역할이 있습니다.
* 테스트할 기능 플래그를 만들었습니다. [첫 번째 기능 플래그 만들기](create-your-first-feature-flag.md)를 참조하십시오.

## 1단계: 여러 변형이 있는 기능 그룹 만들기 {#create}

1. **기능 테스트 > 기능 그룹**(으)로 이동한 다음 **새 기능 그룹**&#x200B;을 선택합니다.
2. **기본 정보**&#x200B;에서 제목, 키 및 설명을 입력하십시오.
3. **백분율 롤아웃**&#x200B;을 설정하여 대상자가 테스트에 얼마나 참여하는지 정의하세요.
4. **Variants**&#x200B;을(를) 1보다 큰 값으로 설정합니다(예: 클래식 A/B 테스트의 경우 2개의 변형).
5. 노출 비율이 여러 변형에 어떻게 분포되어 있는지 이해하려면 [기능 그룹을 점진적으로 롤아웃하도록 설정](set-feature-group-gradual-rollout.md)을 참조하십시오.

## 2단계: 대상자 설정 {#audience}

**대상** 탭에서 대상 기준을 추가하고 포함할 응용 프로그램을 선택합니다. 기능 그룹은 동일한 팀 내에서 여러 애플리케이션을 포괄할 수 있습니다.

>[!NOTE]
>
>A/B 테스트에서 외부 사용자를 타깃팅하려면 **제품 릴리스 소유자** 역할이 있어야 합니다. 개발자 역할은 샌드박스로 제공되며 비공개 테스트로 제한됩니다.

## 3단계: 변형당 기능 추가 {#features}

**기능** 탭 아래에 각 변형의 탭이 있습니다. 각 변형에 적절한 기능 플래그를 추가하여 비교하려는 다양한 경험을 정의합니다.

>[!IMPORTANT]
>
>기능 플래그는 한 번에 하나의 방법(기능 플래그, 기능 그룹 또는 릴리스)을 통해서만 제공될 수 있습니다. 기능 그룹에 플래그를 추가하면 플래그에 직접 설정된 대상자 또는 백분율 롤아웃이 제거됩니다.

## 4단계: 저장 및 활성화 {#save}

기능 그룹 설정을 저장합니다. 테스트를 시작할 준비가 되면 피쳐 그룹을 활성 상태로 설정합니다.
