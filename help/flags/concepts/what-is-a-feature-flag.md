# 기능 플래그란 무엇입니까 {#what-is-a-feature-flag}

기능 플래그는 코드를 재배포하지 않고도 런타임 시 응용 프로그램의 기능을 켜거나 끌 수 있는 메커니즘입니다.

기능 플래그는 **기능 가용성**&#x200B;에서 **코드 배포**&#x200B;를 분리합니다. 플래그 뒤에 숨겨진 기능을 사용하여 새 코드를 프로덕션에 전달한 다음 모든 사용자 또는 타겟팅된 하위 세트에 대해 준비가 되면 언제든지 켤 수 있습니다.

이러한 분리는 위험을 크게 줄입니다. 개발자는 업무 중단을 최소화하면서 지속적으로 구축 및 배송할 수 있으며, 제품 팀은 언제 누구에게 기능이 표시되는지 완벽하게 제어할 수 있습니다.

>[!NOTE]
>
>플래그에서 기능 플래그는 기능 제어의 가장 작은 단위입니다. 단일 기능을 타깃팅하는 데 단독으로 사용하거나 [기능 그룹](feature-groups-to-control-multiple-features.md)의 다른 플래그와 결합할 수 있습니다.
