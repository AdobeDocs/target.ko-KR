# 플래그 모드 {#modes}

플래그는 두 가지 서로 다른 기능 타깃팅 모드를 제공합니다. 각각은 다양한 릴리스 제어 목적을 제공하며 특정 유형의 애플리케이션이나 사용 사례에 맞게 설계되었습니다.

>[!TIP]
>
>제어 중인 사항을 기반으로 모드를 선택합니다(개별 사용자 세션 또는 조직 및 환경 범위).

## 사용자 수준 타기팅 {#user-level}

### 개요 {#user-level-overview}

사용자 수준 타깃팅을 사용하면 개별 사용자 수준에서 기능을 롤아웃할 수 있습니다. 특정 사용자, 내부 테스터, 카나리아 사용자 및 비율 기반의 점진적 롤아웃에 대한 정확한 타겟팅을 지원합니다.

이 모드는 로그인한 사용자에 따라 경험을 변경해야 하는 애플리케이션에 가장 적합합니다.

### 사용 시기 {#user-level-when}

다음 작업을 원할 때 사용자 수준 타깃팅을 사용하십시오.

* 특정 사용자에 대한 기능 활성화
* A/B 실험 실행
* 더 넓은 롤아웃 전에 소규모 그룹으로 카나리아 테스트 수행
* 사용자 비율에 따라 기능을 점진적으로 롤아웃합니다.
* 개별 사용자 특성(이메일 도메인 또는 프로필 데이터 등)을 기반으로 경험을 미세 조정합니다

### 제한 {#user-level-limitations}

사용자 수준 타깃팅은 다음 시나리오에 맞게 디자인되지 않았습니다.

* 조직, 환경 또는 지역 수준에서 기능을 제어하지 않음
* 테넌트 전체 기능 지원 또는 자격 부여를 위한 것이 아닙니다.

## 조직 및 환경 수준 타기팅 {#org-level}

### 개요 {#org-level-overview}

조직 수준 타겟팅을 사용하면 조직, 환경 및 지역 범위에서 기능 롤아웃을 수행할 수 있습니다. 개별 사용자가 아닌 전체 조직 또는 특정 배포 환경에서 기능 가용성을 제어합니다.

이 모드는 엔터프라이즈급 기능 게이트 및 환경별 롤아웃을 위해 설계되었습니다.

### 사용 시기 {#org-level-when}

다음 작업을 원할 때 조직 수준 타깃팅을 사용하십시오.

* 전체 조직에 대한 기능 활성화
* 환경에 따라 기능 가용성 제어(예: 스테이징 및 프로덕션)
* 지역 롤아웃 수행
* 자격 또는 구독 계층에 따른 게이트 기능

### 제한 {#org-level-limitations}

조직 및 환경 수준 타깃팅은 다음 시나리오에 맞게 디자인되지 않았습니다.

* 리치 사용자 프로필 기반 타깃팅을 지원하지 않습니다.
* 개별 사용자 수준에서 백분율 기반 롤아웃을 지원하지 않음
* 실험 또는 A/B 테스트를 위한 것은 아닙니다.

## 비교 {#comparison}

| 차원 | 사용자 수준 타기팅 | 조직 및 환경 수준 타기팅 |
|---|---|---|
| 컨트롤 범위 | 개별 사용자 | 조직, 환경, 지역 |
| 타겟팅 기준 | 사용자 프로필 및 속성 | 조직 및 환경 컨텍스트 |
| 롤아웃 비율 | 예 | 아니오 |
| A/B 테스트 | 예 | 아니오 |
| 엔터프라이즈 게이트 | 아니오 | 예 |
