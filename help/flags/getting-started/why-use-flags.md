# 플래그를 사용하는 이유 {#why-use}

플래그는 기능을 보는 사람과 개발 상태에서 테스트하거나, 사용자 하위 집합으로 검증하거나, 여러 팀에 걸쳐 큰 릴리스를 조정하는 경우 등을 제어해야 할 때마다 올바른 도구입니다.

## 일반적인 사용 사례 {#use-cases}

개발 중 **선택적 테스트**
다른 개발자가 영향을 받지 않고 작업을 계속하는 동안 자체 세션에 대해 기능을 테스트합니다. 복잡한 코드 분기는 필요하지 않습니다.

사용자 피드백이 있는 **준비된 롤아웃**
먼저 기능을 소규모 사용자 그룹에 릴리스합니다. 피드백을 수집하고 문제를 해결한 다음 전체 릴리스 전에 대상자를 점차적으로 확장합니다.

**프로덕션으로의 점진적 롤아웃**
사용자의 1%, 10%, 50%, 100% 순으로 점진적으로 새로운 기능을 엽니다. 각 단계에서 성능 및 사용자 응답을 모니터링합니다. 문제가 발생하면 즉시 끄십시오.

**백 엔드 로드 관리**
모든 사용자를 한 번에 새로운 기능에 노출시키는 대신 백엔드 서비스의 갑작스러운 트래픽 스파이크를 방지하기 위해 롤아웃을 단계적으로 수행합니다.

**조정된 다중 응용 프로그램 릴리스**
특정 사용자 집합에 대해 여러 애플리케이션 및 팀에서 동시에 기능을 사용할 수 있도록 합니다. 플래그는 전체 릴리스 표면에 걸쳐 일관성을 보장합니다.

**지연된 릴리스**
미리 프로덕션에 코드를 배포한 다음, 마지막 코드 변경 없이 정확한 순간(예: 제품 실행 이벤트가 시작될 때)에 기능을 활성화합니다.

**중지 스위치**
릴리스 후 문제가 발견되면 핫픽스 또는 재배포 없이 기능을 즉시 끕니다.
