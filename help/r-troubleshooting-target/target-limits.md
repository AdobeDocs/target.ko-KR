# 제한{#limits}

Adobe Target의 활동 및 기타 요소에 영향을 미치는 문자 제한 및 기타 제한(오퍼 크기, 대상, 프로필, 값, 매개 변수 등)에 대한 정보입니다.

아래 나열된 제한 사항은 권장 제한 사항입니다. 이 제한 사항에 가까워지거나 초과하면, 성능이 느려질 수 있습니다. 하나의 활동에 많은 대상, 타겟 및 경험이 모두 있는 경우와 같이 활동이 매우 복잡해도 인터페이스 로드가 느려질 수 있습니다.

매우 복잡한 활동은 Adobe 컨설턴트와 함께 검토해야 하며 프로덕션 상태에 들어가기 전에 제한된 환경에서 테스트해야 합니다.

## 활동 이름

**제한**: 250 자.

## 대상 이름

**제한**: 256 자.

256자보다 긴 값은 잘립니다.

## categoryId 매개 변수

**제한**: 250 자.

## 고객 속성 이름

**제한**: 128 자.

## 엔티티 사용자 지정 속성

**제한**: 최대 길이는 언어에 따라 다릅니다.

* 15,000 자 (단일 값, 1 바이트 및 2 바이트 언어)
* 500 개 값, 값당 100 자 (다중 값)

단일 값 엔티티 사용자 정의 속성의 최대 길이는 15,000 자 (영어 및 기타 라틴어 스크립트 알파벳과 같은 1 바이트 및 2 바이트 UTF -8 인코딩 언어) 또는 10,000 자 (중국어, 일본어 및 한국어 등의 3 바이트 UTF -8 인코딩 언어의 경우) 입니다.

다중 값 엔티티 사용자 지정 속성은 500 개 이하의 값을 포함할 수 없습니다. 각 개별 값은 100 자로 제한됩니다. 모든 값에 걸친 총 문자 수는 단일 값 엔티티 사용자 지정 속성의 최대 길이에 대한 제한을 준수해야 합니다 (위 참조).

## Entityid 매개 변수

**제한**: 1,000 자.

## Excludedid {#excludedid}

**제한**: 게시물 요청에 대한 5 KB. 2,083 자에서 GET 요청에 대한 URL의 길이를 뺀 것입니다.

GET 요청의 경우, 백엔드의 제한은 5 KB 이지만 Microsoft Internet Explorer가 URL를 2,083 자로 제한한다는 사실 때문에 실제 제한은 2,083 자에서 URL의 현재 길이를 뺀 것입니다.

## 경험 이름

**제한**: 20 자.

## mbox 내부 프로필 속성 값

**제한**: 256 자.

이 값보다 긴 값은 잘립니다.

## mbox 요청에서 mbox 내부 프로필

**제한**: 50 개 프로파일.

50개 이후의 모든 프로필은 무시됩니다.

## mbox 내부 프로필 이름

**제한**: 128 자.

## mbox 이름

**제한**: 250 자.

## mbox 매개 변수

**제한**: mbox 매개 변수에 적용되는 제한 사항은 다음과 같습니다.

* mbox 매개 변수: mbox 당 500 개의 매개 변수.
* 프로필 매개 변수: 500 개의 매개 변수.
* mbox당 프로필 매개 변수:
* 기타 매개 변수(URL, 참조 URL 등): 각 매개 변수 유형에 대해 mbox당 50에서 보냅니다.

Target 데이터베이스에 기록되는 매개 변수의 경우 위의 제한은 표준 mbox 요청용입니다. 웹 브라우저 제한으로 인해 요청이 축소되지 않는 한 이러한 제한이 적용됩니다.

모바일 서비스 SDK에서 [배치 배달 API](https://developers.adobetarget.com/api/#server-side-batch-delivery)를 사용하는 경우 50개의 mbox 매개 변수, 50개의 프로필 매개 변수 및 50개의 다른 매개 변수 유형에 대한 제한은 API 자체의 제한입니다. 배치 배달 API를 사용하여 이들 번호를 더 많이 포함하는 요청을 보낼 수 없습니다. 요청에 이보다 더 많은 제한이 포함된 경우 API가 다음 오류 메시지를 반환합니다.&quot;mboxParameters 수는 100을 초과할 수 없습니다.&quot;

## mbox 요청 URL

**제한**: 2,083 자.

이 제한 사항은 Microsoft Internet Explorer URL 길이 제한 때문입니다.

## mbox3rdPartyId 매개 변수

**제한**: 60 자.

## 오퍼 이름

**제한**: 250 자.

## 오퍼 크기

**제한**: 오퍼에 적용되는 크기 제한은 다음과 같습니다.

* HTML 오퍼: 256KB.
* UI의 시각적 오퍼: 64KB.
* API: 512KB.

글로벌 mbox를 사용하는 경우 이 제한은 페이지에 대해 반환되는 전체 컨텐츠 세트에 대한 것입니다. 오퍼 크기를 제한하면 페이지 로드 시간이 향상됩니다. 이 제한을 초과하는 경우 다음 메시지가 나타납니다.

&quot;경험에 대한 컨텐츠가 너무 커서 전달할 수 없습니다. 더 적은 페이지 코드에 영향을 미치도록 경험을 수정하십시오.&quot;

## orderId 매개 변수

**제한**: 120 자.

권장 제한.

## orderTotal 매개 변수

**제한**: 120 자.

권장 제한.

## productPurchasedId 매개 변수

**제한**: 쉼표로 구분된 값당 47 자.

시스템이 더 이상 모든 것이 잘립니다.

## 재사용 가능 대상/계정

**제한**: 고객 75 명

권장 제한. 너무 많을 경우 인터페이스에서 JavaScript 시간 초과가 발생합니다.

## 타겟 UI의 스크립트 프로필 입력 상자

**제한**: 2,000 자.

권장 제한. 인코딩된 문자열의 크기에 따라 다릅니다. 인코딩된 문자열은 원시 문자열보다 훨씬 더 길 수 있습니다. 문자열이 너무 큰 경우 Adobe Target에 도착하기 전에 실패합니다.

## 스크립트 프로필 이름

**제한**: 50 자.

## 스크립트 프로필 값

**제한**: 2,048 자.

성능상의 이유로 256자를 넘지 않는 반환 값이 좋습니다.

문자열 반환 값의 경우 반환 값의 크기가 2,048 자를 초과하는 경우 시스템에 의해 스크립트가 비활성화됩니다.

배열 반환 값의 경우, 연결된 배열 값의 크기가 2,048 자를 초과하는 경우 시스템에 의해 스크립트가 비활성화됩니다.

## 타겟 조건

**제한**: 1,000 개 값.

권장 제한. 타깃팅 텍스트 영역에 있는 줄로 구분된 값의 수를 나타냅니다. 예를 들어 1,000개의 zip 코드를 zip 코드 타겟에 입력하는 경우가 있습니다.
