# [!DNL Target]을(를) Experimentation Accelerator과 통합

Experimentation Accelerator은 관리자가 [!DNL Adobe Target] 작업 영역 활동을 구성하고 애플리케이션에 표시하는 방법을 관리할 수 있도록 도와줍니다. 각 [!DNL Target] 작업 영역을 적절한 Experimentation Accelerator 샌드박스에 매핑하여 팀은 [!DNL Adobe Target] 및 [!DNL Adobe Journey Optimizer]의 실험을 한 곳에서 볼 수 있습니다.

➡️ [Adobe Experimentation Accelerator에 대해 자세히 알아보기](https://experienceleague.adobe.com/ko/docs/experimentation-accelerator/using/overview)

## 시작하기 전에

샌드박스 할당을 설정하기 전에 필요한 권한이 있는지 확인하십시오. Experimentation Accelerator에서 **[!UICONTROL Administration]**&#x200B;에 액세스하려면 **[!UICONTROL Manage configuration]** 권한이 있어야 합니다.

사용자는 두 조건이 모두 충족되는 경우에만 [!DNL Target] 작업 영역을 샌드박스에 할당할 수 있습니다.

* 사용자에게 Experimentation Accelerator에서 **[!UICONTROL Manage configuration]** 권한이 있습니다.
* 사용자가 [!DNL Target] 제품 관리자입니다.

## [!DNL Target] 작업 영역에 대한 샌드박스 할당 설정

작업 영역을 할당하기 전에 동일한 테스트에 대한 중복 항목을 방지하기 위해 [!DNL Target] 작업 영역을 하나의 샌드박스에만 할당할 수 있습니다.

각 [!DNL Target] 작업 영역이 표시되는 샌드박스를 선택하려면 다음을 수행하십시오.

1. Experimentation Accelerator에서 **[!UICONTROL Administration]**&#x200B;을(를) 엽니다.

   ![](assets/experimentation-1.png)

1. 현재 [!DNL Target] 작업 영역과 샌드박스 간 할당 표를 검토하십시오.

   ![](assets/experimentation-2.png)

1. **[!UICONTROL Edit]** 아이콘을 클릭합니다.

   ![](assets/experimentation-3.png)

1. 각 샌드박스에 대해 적절한 [!DNL Target] 작업 공간을 할당합니다.

   ![](assets/experimentation-4.png)

1. 변경 내용을 적용하려면 **[!UICONTROL Save]**&#x200B;을(를) 클릭하십시오.

[!DNL Target] 작업 영역에 대한 초기 연결이 만들어지면 시스템에서 업데이트가 전파될 수 있도록 최대 30분을 허용합니다.
