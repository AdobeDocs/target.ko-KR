## 이 유형의 활동을 사용하는 이유는 무엇입니까? {#section_46A70DD7CE3448749E635DDF5EAFC131}

| 활동 유형 | 이유 |
|--- |--- |
| 수동 [!UICONTROL A/B 테스트] | 규칙보다는 비율에 따라 분할하여 트래픽을 강력한 제어 방식으로 측정함으로써 테스트 데이터를 분석하고 대상에 대한 명확한 정보를 얻으며 가장 높은 성과를 나타내는 경험을 파악할 수 있습니다. |
| [!UICONTROL 자동 할당] | 우승 경험을 식별하고 트래픽 할당을 조정하여 우승 경험을 가능한 한 빨리 방문자에게 전달하여 더 빠르고 더 높은 전환 가능성을 지원하는 방법입니다. |
| [!UICONTROL 자동 타겟팅] | 여러 경험 사이에서 승자를 식별한 후 특정 방문자에게 가장 적합한 경험을 전달하는 방법입니다. 이 알고리즘에서는 특정 시간에 특정 경험에 대한 방문자의 전환 성향을 예측하므로 시간이 지남에 따라 방문자의 관심사가 변하면서 타기팅도 조정됩니다. |
| [!UICONTROL Automated Personalization] (AP) | 오퍼 세트(단일 페이지의 요소에서 또는 여러 페이지 간에 만들어지거나 미리 정의됨)를 개인화하고 특정 방문자를 가장 잘 끌어오는 오퍼 조합을 전달하는 방법입니다. |
| [!UICONTROL Multivariate Testing] (MVT) | 여러 오퍼를 여러 요소에 표시한 다음, 특정 목표에 대해 결과 고유 경험을 동시에 테스트하여 가장 성공적인 요소 변형을 결정하는 방법입니다. MVT 활동은 방문자의 상호 작용에 가장 큰 긍정적 또는 부정적인 영향을 미치는 요소를 보여줄 수도 있습니다. |
| [!UICONTROL 경험 타겟팅] (XT) | 정의된 할당 규칙 세트를 기준으로 특정 콘텐츠를 특정 대상으로 타깃팅하는 방법입니다. |

## 어떤 유형의 활동을 사용해야 합니까? {#section_A843D663D3E543FFB1A594266B560395}

| 활동 유형 | 마케터 |
|--- |--- |
| 수동 [!UICONTROL A/B 테스트] | 통계에 정통합니다.<P>테스트 기간이 끝날 때까지 대기했다가 결과를 분석할 시간이 있습니다. |
| [!UICONTROL 자동 할당] | 기간이 짧습니다.<P>최상의 경험을 파악하고 신속하게 제공해야 합니다.<P>테스트 실행 시 결과를 &quot;빠르게 확인&quot;하려고 합니다. |
| [!UICONTROL 자동 타겟팅] | 몇 가지 적합한 경험이 있습니다.<P>동적 및 변화하는 프로필을 기준으로 최적 시간에 특정 방문자에게 경험을 일치시키려고 합니다. |
| [!UICONTROL Automated Personalization] (AP) | 하나 이상의 오퍼가 있습니다.<P>다양한 고유 프로필 및 동작을 통해 특정 방문자를 위한 개인화된 최적 경험을 생성하는 오퍼 조합을 만들려고 합니다. |
| [!UICONTROL Multivariate Testing] (MVT) | 통계에 정통합니다.<P>하나 이상의 오퍼가 있습니다.<P>페이지 요소 상호 작용과 관련된 전환 트렌드를 분석하려고 합니다. |
| [!UICONTROL 경험 타겟팅] (XT) | 특정 대상에게 특정 경험이나 콘텐츠를 제공해야 합니다. |

## 통계적 세부 정보 {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| 활동 유형 | 세부 사항 |
|--- |--- |
| 수동 [!UICONTROL A/B 테스트] | 이 테스트에서는 각 챌린저 경험을 제어 경험과 비교한 다음 모든 경험의 성과를 순위를 매겨, 제어와 비교할 때 가장 성과가 좋은 경험과 성과가 나쁜 경험을 모두 식별합니다. |
| [!UICONTROL 자동 할당] | 이 테스트는 진정한 승자에 대한 통계적 보증을 바로 생성한 후, 이러한 가장 성과가 좋은 경험으로 전환할 가능성이 더 높은 대상 쪽으로 더 많은 트래픽을 전송합니다. |
| [!UICONTROL 자동 타겟팅] | 최적화 메커니즘은 어떤 경험을 어떤 방문자에게 전달할지 결정하기 전에, 그리고 시간에 따른 상승도의 증감을 보여줌으로써 각 경험에 대한 관련 대상을 식별합니다. 최적화 메커니즘은 전환, 세그먼트, 매개 변수 및 프로필 스크립트를 통해 알려집니다. 거기에서 메커니즘은 더 높은 상승도와 전환율을 생성하기 위해 사용할 알고리즘을 자동으로 선택합니다. |
| [!UICONTROL Automated Personalization] (AP) | 오퍼의 성과가 동시 통제군을 기준으로 측정되면서, 최적화 메커니즘은 새 방문자의 행동 및 비슷한 방문자의 과거 행동을 기준으로 어떤 방문자에게 어떤 경험을 전달할지를 지속적으로 조정합니다. |
| [!UICONTROL Multivariate Testing] (MVT) | 이 테스트는 특정 요소가 전환에 미치는 상대적인 영향을 알아내는 데 도움이 됩니다. |
| [!UICONTROL 경험 타겟팅] (XT) | 이 방법은 특정한 경험 또는 특정 콘텐츠를 특정 대상에게 타기팅하는 규칙을 정의합니다. 고객은 경험 수준에서 업데이트할 수 있습니다. |

## 이점 및 고려 사항 {#section_56C46ABEF7B945DDA0C1E6D714377123}

| 활동 유형 | 이점 | 고려 사항 |
|--- |--- |--- |
| 수동 [!UICONTROL A/B 테스트] | A/B 휴식은 어떤 경험이 가장 성과가 좋은지 뿐만 아니라, 각 경험이 어떻게 성과를 내는지에 대해 완전히 이해하는 데 도움이 됩니다. | 다음에서 [!UICONTROL A/B 테스트], 샘플 크기에 도달하기 전에 테스트 결과를 보는 경우 부정확한 결과에 의존할 위험이 있습니다(&quot;이전&quot;을 &quot;미리 볼 수 없습니다!).<P>좋아하지 않음 [!UICONTROL 자동 할당], A/B 테스트에서 트래픽 분포는 일부 경험이 다른 경험보다 뛰어난 성과를 보인다는 사실을 인식하더라도 고정된 상태를 유지합니다.<P>의 모범 사례에 대한 정보 [!UICONTROL A/B 테스트] 활동, 참조 [A/B 테스트를 얼마 동안 실행해야 합니까?](/help/main/c-activities/t-test-ab/sample-size-determination.md) 및 [10가지 일반적인 A/B 테스트 위험 및 이를 피하는 방법](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL 자동 할당] | [!UICONTROL 자동 할당은 수동 A/B 테스트보다 전반적으로 더 높은 전환율을 유지하므로 일반적인 A/B 테스트 비용을 절감합니다. ] 다음과 같은 이유로 전환율이 더 높습니다. [!UICONTROL 자동 할당] 더 많은 트래픽을 가장 성과가 좋은 경험으로 푸시합니다. 즉, 테스트 기간이 종료되기 전에 해당 우승 경험의 이점을 미리 실감할 수 있습니다(엿볼 수 있음). | [!UICONTROL 자동 할당은 승자를 식별하지만 패자 간의 구분하지 않습니다. ] 각 경험의 수행 방법을 알고 있어야 하는 경우에는 A/B 테스팅을 사용하는 것이 좋습니다.<P>다음 [!UICONTROL 자동 할당] 이 기능은 고급 지표 설정인 &quot;증분 카운트 및 사용자를 활동에 유지&quot;에 따라서만 작동합니다. 반복 전환을 카운트하지 않으려면 A/B 테스트를 대신 사용해야 합니다. |
| [!UICONTROL 자동 타겟팅] | 포함 [!UICONTROL 자동 타기팅], 다중 페이지 경험을 포함한 모든 종류의 경험에 머신 러닝이 적용됩니다. An [!UICONTROL 자동 타기팅] 활동을 사용하면 다음 값을 얻을 수도 있습니다. [!UICONTROL Automated Personalization] 익숙한 A/B 테스트 워크플로를 사용하는 동안. | 포함 [!UICONTROL 자동 타기팅], 오퍼의 콘텐츠를 자주 또는 자주 변경하려는 경우 알고리즘은 각 변경 후에 학습한 내용을 활용하고 해당 콘텐츠를 적절한 방문자에게 전달할 충분한 시간이 필요합니다. |
| [!UICONTROL Automated Personalization] (AP) | 포함 [!UICONTROL Automated Personalization]를 사용하면 한 곳에서 모든 오퍼를 수집할 수 있으며 알고리즘이 오퍼의 최상의 조합을 결정합니다. 개별 경험을 지정하거나 빌드할 필요가 없습니다. [!UICONTROL Automated Personalization] 과 동일한 머신 러닝 알고리즘을 사용합니다.
