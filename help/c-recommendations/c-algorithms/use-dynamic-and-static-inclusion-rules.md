# ![PREMIUM](/help/assets/premium.png) 동적 및 정적 포함 규칙 사용

[!DNL Adobe Target]에서 기준 및 프로모션에 대한 포함 규칙을 만들고, 동적 또는 정적 필터링 규칙을 추가하여 권장 사항에 대한 더 나은 결과를 획득하는 방법에 대한 정보입니다.

기준 및 프로모션에 대한 포함 규칙을 만들고 사용하는 프로세스는 사용 사례 및 예와 비슷합니다. 기준 및 프로모션 및 포함 규칙 사용은 모두 이 섹션에서 다룹니다.

## 기준에 필터링 규칙 추가 {#section_CD0D74B8D3BE4A75A78C36CF24A8C57F}

[기준을 만들려면](/help/c-recommendations/c-algorithms/create-new-algorithm.md#task_8A9CB465F28D44899F69F38AD27352FE) **[!UICONTROL 포함 규칙]** 아래의 **[!UICONTROL 필터링 규칙 추가]**&#x200B;를 클릭하십시오.

![](assets/inclusion_options_new.png)

사용 가능한 선택 사항은 선택한 업계 카테고리와 권장 사항 키에 따라 다릅니다.

## 프로모션에 필터링 규칙 추가 {#section_D59AFB62E2EE423086281CF5D18B1076}

[프로모션을 만들려면](/help/c-recommendations/t-create-recs-activity/adding-promotions.md#task_CC5BD28C364742218C1ACAF0D45E0E14) **[!UICONTROL 속성별 프로모션]**&#x200B;을 선택한 다음, **[!UICONTROL 필터링 규칙 추가]**&#x200B;를 클릭하십시오.

![](assets/inclusion_options.png)

## 필터 유형 {#section_0125F1ED10A84C0EB45325122460EBCD}

다음 섹션에서는 기준 및 프로모션에 대해 [!UICONTROL 동적 필터링] 및 [!UICONTROL 값으로 필터링 선택 사항 유형을 나열합니다.]

### 동적 필터링

동적 포함 규칙은 정적 포함 규칙보다 더 강력하며 더 나은 결과 및 참여를 제공합니다. 다음 사항을 고려하십시오.

* 동적 포함 규칙은 사용자의 프로필 매개 변수 또는 mbox 호출에 있는 속성을 일치시켜 권장 사항을 제공합니다.

   예를 들어 &quot;가장 방문 빈도가 높은 기준&quot; 권장 사항을 만들 수 있습니다. 반환된 권장 사항 집합에서 사용자가 권장 사항이 표시된 페이지에 액세스할 때 전달된 속성에 대해 권장 사항(실시간)을 필터링할 수 있습니다.

* 정적 규칙을 사용하여 컬렉션을 사용하는 대신 권장 사항에 포함된 항목을 제한합니다.

* 필요한 만큼 동적 포함 규칙을 만들 수 있습니다. 포함 규칙들은 AND 연산자로 결합됩니다. 권장 사항에 항목을 포함하려면 모든 규칙을 충족해야 합니다.

동적 필터링에 사용할 수 있는 옵션은 다음과 같습니다.

| 동적 필터링 옵션 | 세부 사항 |
| --- | --- |
| [엔티티 속성 일치](/help/c-recommendations/c-algorithms/entity-attribute-matching.md) | 잠재적 권장 사항 항목의 풀을 사용자가 상호 작용한 특정 항목과 비교하여 동적으로 필터링합니다.<br>방문자 [!UICONTROL 의 ] 즐겨찾기 브랜드와 같이 방문자에게 가장 잘 맞는 추천을 표시하려면 엔티티 속성 일치 를 사용하십시오. |
| [프로필 속성 일치](/help/c-recommendations/c-algorithms/profile-attribute-matching.md) | 항목(엔티티)을 사용자 프로필에 있는 값과 비교하여 동적으로 필터링합니다.<br>크기 또 [!UICONTROL 는 ] 즐겨찾기 브랜드와 같이 방문자 프로필에 저장된 값과 일치하는 권장 사항을 표시하려면 프로필 속성 일치 를 사용합니다. |
| [매개 변수 일치](/help/c-recommendations/c-algorithms/parameter-matching.md) | 항목(엔티티)을 요청(API 또는 mbox)에 있는 값과 비교하여 동적으로 필터링합니다.<br>매개  [!UICONTROL 변수 ] 일치 를 사용하여 페이지 매개 변수 또는 장치 차원이나 지리적 위치 등의 방문자 매개 변수와 일치하는 컨텐츠를 추천합니다. |

### 값별 필터링

값별로 필터링할 다음 옵션을 사용할 수 있습니다.

## 사용 가능한 연산자 {#operators}

동적 기준 및 프로모션은 정적 기준 및 프로모션보다 훨씬 강력하며 더 나은 결과 및 참여를 제공합니다.

다음 예에서는 마케팅 작업에서 동적 프로모션 및 제외를 사용하는 방법에 대한 일반적인 아이디어를 제공합니다.

| 연산자 | 예 |
| --- | --- |
| Equals<br>(엔티티 속성 일치, 프로필 속성 일치, 매개 변수 일치 및 정적 필터에서 사용할 수 있습니다.) | 동적 프로모션에서 &quot;equals&quot;(다음과 같음) 연산자를 사용할 경우, 방문자가 웹 사이트에서 항목을 보고 있으면(제품, 문서 또는 동영상 등) 다음 방법으로 다른 항목을 프로모션할 수 있습니다.<ul><li>동일한 브랜드</li><li>동일한 카테고리</li><li>주택 브랜드의 동일한 카테고리 AND</li><li>같은 가게</li></ul> |
| Does Not Equal<br>(엔티티 속성 일치, 프로필 속성 일치, 매개 변수 일치 및 정적 필터에서 사용할 수 있음) | 동적 프로모션에서 &quot;does not equal&quot;(다음과 같지 않음) 연산자를 사용할 경우, 방문자가 웹 사이트에서 항목을 보고 있으면(제품, 문서 또는 동영상 등) 다음 방법으로 다른 항목을 프로모션할 수 있습니다.<ul><li>다른 TV 시리즈</li><li>다른 장르</li><li>다른 제품 시리즈</li><li>다른 스타일 ID</li></ul> |
| Substring<br>(엔티티 속성 일치, 프로필 속성 일치, 매개 변수 일치 및 정적 필터에서 사용할 수 있음)을 포함하지 않습니다. | 방문자가 웹 사이트에서 항목을 보고 있는 경우(예: 제품) &quot;하위 문자열 포함하지 않음&quot; 연산자를 사용하여 다음과 같은 다른 항목을 프로모션할 수 있습니다.<ul><li>제목에 욕어가 없습니다.</li></ul> |
| <br>으로 시작합니다(엔티티 속성 일치, 프로필 속성 일치, 매개 변수 일치 및 정적 필터에서 사용 가능). | 다음으로 시작 연산자를 사용하는 경우, 방문자가 웹 사이트에서 항목을 보고 있으면(제품 등) 다음과 같은 다른 항목을 프로모션할 수 있습니다.<ul><li>제품 이름은 iPhone으로 시작</li></ul> |
| <br>(엔티티 속성 일치, 프로필 속성 일치, 매개 변수 일치 및 정적 필터에서 사용 가능)으로 끝납니다. | 다음으로 끝남 연산자를 사용하는 경우, 방문자가 웹 사이트에서 항목을 보고 있으면(제품 등) 다음과 같은 다른 항목을 프로모션할 수 있습니다.<ul><li>콘텐츠는 영어 언어를 나타내는 EN으로 끝납니다</li></ul> |
| 이<br>보다 크거나 같음(엔티티 속성 일치, 프로필 속성 일치, 매개 변수 일치 및 정적 필터에서 사용 가능) | 방문자가 웹 사이트에서 항목을 보고 있는 경우(예: 제품) &quot;다음보다 크거나 같음&quot; 연산자를 사용하여 다음과 같은 다른 항목을 프로모션할 수 있습니다.<ul><li>가격이 동일하거나 더 비싸다</li></ul> |

| Is Between<br>(엔티티 속성 일치, 프로필 속성 일치 및 매개 변수 일치에서 사용 가능) | 동적 프로모션에서 &quot;is between&quot;(다음 사이) 연산자를 사용할 경우, 방문자가 웹 사이트에서 항목을 보고 있으면(제품, 문서 또는 동영상 등) 다음과 같은 다른 항목을 프로모션할 수 있습니다.<ul><li>더 비싼 값</li><li>저렴한 가격</li><li>비용 + 또는 마이너스 30%</li><li>같은 시즌의 나중 에피소드</li><li>연속된 이전 책</li></ul> |
| List<br>에 포함되어 있습니다(프로필 속성 일치 및 매개 변수 일치에서 사용할 수 있음). | 프로필 속성 일치에서 &quot;is contains in list&quot; 연산자를 사용하여 방문자가 웹 사이트에서 항목을 보고 있으면(제품, 문서 또는 동영상 등) 다음과 같은 다른 항목을 프로모션할 수 있습니다.<ul><li>방문자의 지리적 위치에서 사용할 수 있습니다</li></ul>**예**:방문자의 지리적 영역에서 사용할 수 있는 항목만 추천하려고 합니다.<br>필터 규칙은 <br>`availableGeographies list contains an item in user.currentGeography`<br>**참고**:이 연산자를 사용할 때 규칙  [오른쪽](#caveats) 에 목록이 필요합니다. |

## 엔티티 속성 일치, 프로필 속성 일치 및 매개 변수 일치로 필터링할 때 빈 값 처리 {#section_7D30E04116DB47BEA6FF840A3424A4C8}

종료 기준 및 프로모션에 대해 [!UICONTROL 엔티티 속성 일치], [!UICONTROL 프로필 속성 일치] 및 [!UICONTROL 매개 변수 일치]를 기준으로 필터링할 때 빈 값을 처리할 수 있는 옵션을 여러 개 선택할 수 있습니다.
